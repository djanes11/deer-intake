# Pilot Onboarding Checklist

Use this checklist for every new processor pilot.

## Before You Create The Processor

- Confirm the processor is deer-focused and understands the product scope.
- Confirm which state they operate in.
- Confirm whether they want:
  - manual workflow only
  - texting
  - scanners/labels
- Confirm their preferred pilot start date.
- Confirm the owner/admin contact who will approve setup.

## Platform Setup

- Create the processor record in platform admin.
- Set public hostname.
- Set staff hostname.
- Create or attach the first processor admin login.
- Confirm the processor is marked active only when setup is ready.

## Processor Setup

- Add business/public name.
- Add address, maps link, and phone.
- Add hours.
- Add pricing.
- Review process types.
- Review add-ons.
- Review specialty products.
- Review customer-facing copy.
- Review notification templates.
- Confirm state form selection.

## Workflow Setup

- Decide whether public intake is enabled.
- Decide whether scan workflow is enabled.
- Decide whether cape scan workflow is enabled.
- Decide whether SMS is enabled.
- Confirm tag format and starting tag number.
- Confirm accepted payment methods.
- Confirm hold/storage policy wording.

## Hardware Check

- If using standard printers only: verify intake sheet prints.
- If using thermal labels: verify label output.
- If using scanners: verify a live scan moves a deer correctly.
- Confirm fallback plan if scanner/printer is unavailable.

## Live Pilot Verification

- Submit one public intake from a phone.
- Assign a permanent tag from staff workflow.
- Print intake sheet.
- Search and reopen the deer.
- Progress one deer through scan or manual status flow.
- Verify status lookup works from customer side.
- Send one test notification.
- Record one payment and mark one pickup handoff.

## Go/No-Go

Go live only if all are true:

- owner/admin can sign in
- public intake works
- staff search works
- print path works
- status lookup works
- at least one notification path is confirmed
- fallback plan is documented for the shop

If any of the above fail, pause the pilot and fix them before handing the system to daily staff.
